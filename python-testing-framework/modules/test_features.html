test_features.py#

Overview#

As detailed in Details, this module is used to populate the builtins module in order to fascilitate the custom execution of the test files. The exception is any key mentioned in the NO_EXPORT variable and that variable itself. Currently this module defines a custom implementation of __build_class__ which is called to execute the class body of classes being defined. If Test is a direct base of a class it will be handled by internals.run_test() and in other cases the original __build_class__ will be called using a reference in state.original_builtins.

This module also defines a custom print() function that logs prints to the top most test on the test stack state.test_stack. Prints that happens outside of a runnin test will be printed using the original print function.

The Test class in here is used to identify tests as described above.

This module also defines a function to help with iterating through variables from the calling locals, IterVars.
