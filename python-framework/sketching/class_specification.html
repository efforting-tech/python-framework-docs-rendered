Class specification#

Warning

This section of the documentation relates to sketches of features and does not reflect actually implemented features. The implemented features are available at Modules.

  • Member descriptors

Rationale#

This sketch is trying to answer how to declare classes that makes uses of the type system. The basic idea is that instances of member_descriptor will be utilized as members in class bodies.

#Get shorthand
from efforting.tech.class_specification import member_descriptors as M

#Declare class
class some_class(abc.thing.stuff, abc.other.thing):
        member1 = M.field()
        cache1 = M.cache()

Note

The example above may not represent the actual members of member_descriptors since they are not fully flushed out yet.

The idea here is that the various instances of member_descriptor (M.field() and M.cache()) declares the functions of the various members. These functions will be implemented by the machinery in abc_type and they may even add or remove members. This makes it possible to have features that require multiple members, such a cache value and a data descriptor.
