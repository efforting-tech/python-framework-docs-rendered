class abc_type(name, bases, scope)#
Parameters

  • name (str) – Name of the abc type

  • bases (tuple of abc_type) – Bases for the new abc_type

Warning

This class can be created in two very different manners depending on whether bases is set or not. If there are no bases the returned class will be an abc but if bases are specified the assumption is that a concrete class is to be created. When creating a concrete class there will be a complex machinery in place to assist with a plethora of features related to this.

These features are still in development and as they progress this message will be updated to be less vague.

__getattr__(name)#

Retrieve a base class by name relative to the current one from the _abc_directory. If the current abc_type is path.to.thing and name is item the abc_type for path.to.thing.item is returned.

Raises

AttributeError – if there is such class in the _abc_directory.

__instancecheck__(instance)#
Parameters

instance (object) – An instance - possibly registered as a concrete class of some ABC.

First the bases of instance.__class__ is looked up in the _abc_registry. Then the data_path of the current abc_type is walked and for each subsequent data_path (including the initial one) it is checked whether that candidate is one of the bases. If that is the case True is returned. If no such case is identified, then False is returned.

The purpose of this is to separate this ABC system from the Python built in one.

Return type

bool
